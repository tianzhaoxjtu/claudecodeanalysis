# 仓库分析产物说明

本目录包含两类中文分析产物：

- `代码仓设计与实现深度分析报告.md`
  面向资深工程师的深度 Markdown 报告，强调架构、运行链路、扩展机制、风险与学习路径。
- `learning-site/`
  独立静态学习网站，按“总览 -> 架构 -> 模块 -> 运行 -> 执行流 -> 配置与构建 -> 设计评估 -> 学习路径 -> 关键文件/术语”的方式重组内容。
  站点包含基于真实代码路径绘制的 Mermaid 图，浏览器会在页面端直接渲染，无需构建步骤。

同时保留一层证据笔记：

- `evidence/module-map.md`
- `evidence/entrypoints-and-flows.md`
- `evidence/config-and-extension-notes.md`
- `evidence/design-risks.md`

这些文件用于说明报告和网站的结论依据，重点记录可确认事实、合理推断和未确认项。

## 如何阅读

推荐顺序：

1. 先读 `代码仓设计与实现深度分析报告.md` 的“项目概览”“入口点与生命周期”“关键执行流”。
2. 再打开 `learning-site/index.html`，按学习路径逐页浏览。
3. 如果需要追溯依据，再看 `evidence/` 下的笔记。

## 如何本地预览学习网站

方式一：直接打开 `learning-site/index.html`。

- 学习站中的架构图、依赖图、时序图和数据流图通过浏览器端 Mermaid 运行时渲染。
- 如果浏览器无法加载 Mermaid 运行时，页面会降级显示 Mermaid 源码块，不会出现空白区域。

方式二：在本目录启动一个静态文件服务：

```bash
cd codex_analysis_output/learning-site
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 当前分析边界

- 当前工作目录不是完整 Git 仓库，根目录仅见 `src/` 源码树，未见 `.git`、`package.json`、CI 配置、测试框架配置。
- 因此本次分析严格以可见源码为证据边界。
- 报告中所有结论均尽量标记为：
  - 已确认事实
  - 合理推断
  - 未确认项
